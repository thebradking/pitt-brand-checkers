# Pitt Document Checker

A browser-based tool that checks University of Pittsburgh documents against the **Pitt Brand Guidelines v2.2** and the **Pitt Writing Style Manual (3rd Edition)**. Upload a PDF and get annotated feedback directly on your document — no server, no installation, no data stored anywhere.

![Pitt Brand Colors](https://img.shields.io/badge/Pitt%20Royal-%23003594?style=flat&logoColor=white) ![Pitt Gold](https://img.shields.io/badge/Pitt%20Gold-%23FFB81C?style=flat&logoColor=003594)

---

## What It Checks

**Brand Compliance**
- Color usage (Pitt Royal, Pitt Gold, approved secondary and accent palette)
- Typography (ATF Alternate Gothic, Merriweather, Roboto — flags off-brand fonts)
- Logo and wordmark usage rules
- Messaging guidelines ("Hail to Possible" vs. "It's Possible at Pitt" usage)

**Writing Style (3rd Edition)**
- University name references and capitalization
- Office and department naming (including 3rd Ed. updates)
- Numbers, abbreviations, punctuation rules
- Gender-inclusive language
- Degree formatting, season capitalization, phone number format
- Commonly misused words and Pittsburghese constructions

## Setup: API Key

This tool uses the [Anthropic API](https://www.anthropic.com) (Claude) to analyze your document. You will need your own Anthropic API key.

1. Sign up or log in at [console.anthropic.com](https://console.anthropic.com)
2. Go to **API Keys** and create a new key
3. Paste it into the key field at the top of the tool when you open it

**Your key is stored only in your browser session (sessionStorage).** It is never written to any file, never committed to this repo, and is cleared when you close the tab. It goes directly from your browser to Anthropic — nothing passes through any intermediate server.

> ⚠️ **Never paste your API key into the source code or commit it to this repo.** Anyone who finds a key in a public repo can use it and charge your account. The tool is specifically designed so this never happens.

2. Go to your repo → **Settings** → **Pages**
3. Under "Source," select **Deploy from a branch**
4. Choose **main** branch, **/ (root)** folder, and click Save
5. GitHub will give you a URL at `https://your-username.github.io/repo-name`

That's it. The tool works entirely in the browser — no backend needed.

## How It Works

1. You upload a PDF — it never leaves your browser
2. [PDF.js](https://mozilla.github.io/pdf.js/) renders each page to a canvas element
3. Text is extracted from the PDF's data layer page by page
4. The extracted text is sent to the Anthropic API with the full Pitt brand and style rules embedded in the prompt
5. Claude returns a structured list of issues with page numbers and approximate positions
6. Numbered annotation markers are drawn on top of the rendered PDF pages
7. The sidebar lists all issues — click any to jump to that page and highlight the marker

---

## Limitations

- **Text-layer PDFs only.** Scanned documents (image-only PDFs with no text layer) cannot be analyzed. If your PDF was exported from InDesign, Word, or Google Docs, it will work fine.
- **Color detection is text-based.** The tool analyzes text references to colors and design choices — it does not do pixel-level color scanning of images or graphics.
- **Logo misuse detection is approximate.** Rules like "do not distort the logo" require visual inspection the AI cannot do from text alone.
- **AI analysis may vary.** Results are generated by a language model and should be treated as a starting point for review, not a definitive compliance audit.
